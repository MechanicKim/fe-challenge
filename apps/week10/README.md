# 10주차 챌린지 가이드: 토스트(Toast) 알림 시스템

10주차 챌린지는 사용자에게 현재 상황(성공, 실패, 경고 등)을 **비침습적**으로 알리는 **토스트 알림(Toast Notification) 시스템**을 구축하는 것입니다. 이 시스템은 알림의 큐(Queue) 관리, 자동 소멸, 그리고 유형별 스타일링 능력을 요구합니다.

### 🚀 챌린지 개요: 목표와 요구사항

| 카테고리 | 상세 목표 |
| :--- | :--- |
| **핵심 기능** | 여러 개의 알림을 **순서대로 또는 쌓아서** 표시할 수 있어야 하며, 각 알림은 일정 시간 후 **자동으로 사라져야** 합니다. |
| **주요 기술** | **프레임워크** (선택) 또는 Vanilla JS, CSS Animation, `setTimeout`, 상태 관리(배열). |
| **사용자 경험** | 알림이 나타나고 사라질 때 부드러운 **CSS 애니메이션**을 적용해야 하며, 마우스를 올리면(Hover) **자동 소멸 시간이 일시 정지**되어야 합니다. |
| **재사용성** | 애플리케이션의 어느 컴포넌트에서도 쉽게 호출할 수 있는 **전역적인 함수 또는 훅(Hook)** 형태로 설계되어야 합니다. |

---

### ⚙️ 필수 구현 단계

#### 1. 기본 UI 및 상태 관리

1.  **컨테이너 설정:** 모든 토스트 알림이 표시될 페이지의 고정된 영역(예: 오른쪽 상단)에 **토스트 컨테이너**를 `position: fixed`를 사용하여 배치합니다.
2.  **알림 상태:** 전역 상태(React Context, Vue Pinia/Vuex, 또는 JS 배열)를 사용하여 **현재 활성화된 토스트 알림들의 목록**을 관리합니다. 각 알림은 `id`, `message`, `type`, `duration` 등의 속성을 가져야 합니다.
3.  **렌더링:** 상태 목록의 변화에 따라 토스트 알림 컴포넌트들을 컨테이너 내부에 순서대로 렌더링합니다.

#### 2. 알림 표시 및 소멸 로직 (핵심)

1.  **전역 `showToast()` 함수:** 메시지, 유형, 지속 시간 등을 인수로 받아 새로운 알림 객체를 생성하고 **상태 목록에 추가**하는 함수를 구현합니다.
2.  **자동 소멸:** 알림이 생성될 때, `setTimeout`을 사용하여 지정된 시간 후에 해당 알림의 `id`를 사용하여 상태 목록에서 **제거**되도록 구현합니다.
3.  **유형별 스타일:** 알림 유형(`success`, `error`, `warning`, `info`)에 따라 **아이콘**과 **배경색**이 다르게 표시되도록 CSS를 적용합니다.

#### 3. UX 및 이벤트 처리

1.  **애니메이션:** 새로운 알림이 나타날 때는 부드러운 **슬라이드-인 애니메이션**을, 사라질 때는 **슬라이드-아웃 또는 페이드-아웃 애니메이션**을 적용합니다.
2.  **Hover 일시 정지:** 사용자가 마우스를 토스트 알림 위에 올리면, **자동 소멸 타이머가 일시 정지**되어야 합니다. 마우스가 벗어나면 타이머가 재개됩니다. (재개 시점에 잔여 시간이 아니라 처음부터 다시 시작되지 않도록 정교하게 구현해야 합니다.)
3.  **수동 닫기:** 각 토스트에 작은 'x' **닫기 버튼**을 추가하여 사용자가 수동으로 즉시 닫을 수 있도록 합니다.

### 🌟 심화 도전 과제 (Challenge Up)

실력을 더 높이고 싶다면 다음 항목에 도전해 보세요.

1.  **알림 위치 옵션:** 토스트 알림이 표시되는 위치(상단-오른쪽, 하단-왼쪽 등)를 함수 호출 시 지정할 수 있도록 시스템을 확장합니다.
2.  **콜백(Callback) 지원:** 토스트 알림에 버튼을 추가하고, 이 버튼 클릭 시 특정 함수가 실행되도록 콜백 기능을 구현합니다.
3.  **`Promise` 기반 알림:** 비동기 작업(API 호출 등)의 시작/성공/실패 상태를 `Promise`를 사용하여 자동으로 토스트 알림에 연결하여 표시하는 기능을 구현합니다.
