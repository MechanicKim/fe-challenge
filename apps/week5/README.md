# 5주차 챌린지 가이드: 커스텀 폼 유효성 검사 라이브러리

5주차 챌린지는 프론트엔드 개발의 기초이자 핵심인 **폼(Form) 처리의 확장성과 재사용성**을 다룹니다. 프레임워크에 의존하지 않는 **Vanilla JavaScript**를 사용하여, 복잡한 규칙을 중앙에서 관리하는 **유효성 검사 코어**를 구축하는 것이 목표입니다.

### 🚀 챌린지 개요: 목표와 요구사항

| 카테고리 | 상세 목표 |
| --- | --- |
| **핵심 기능** | 폼 데이터를 수집하고, **JavaScript 객체 스키마**에 정의된 규칙을 기반으로 유효성 검사를 실행하는 재사용 가능한 함수 개발. |
| **주요 기술** | **Vanilla JavaScript**, DOM 조작, 객체 지향 설계(모듈화). |
| **데이터 관리** | 모든 유효성 검사 규칙 및 에러 메시지는 **중앙 집중식 JS 스키마**에 정의되어야 합니다. |
| **UI/UX** | 검사 실패 시, 해당 입력 필드 아래에 **명확한 에러 메시지**를 표시해야 합니다. |

### ⚙️ 필수 구현 단계

### 1. HTML 구조 및 스키마 정의

1. **폼 UI 구축:** 회원 가입 또는 로그인 형태의 간단한 폼을 HTML/CSS로 만듭니다. 각 입력 요소에는 유효성을 식별하기 위한 **고유한 `name` 속성**만 부여합니다. (예: `<input name="username">`)
2. **중앙 스키마 정의:** 폼 전체의 유효성 검사 규칙을 담는 JavaScript 객체(`validationSchema`)를 정의합니다.

**스키마 예시:**

```
const validationSchema = {
    username: {
        required: true,
        minLength: 5,
        messages: {
            required: "사용자 이름은 필수 입력 항목입니다.",
            minLength: "최소 5자 이상 입력해야 합니다."
        }
    },
    userEmail: {
        required: true,
        regex: /^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$/,
        messages: {
            required: "이메일 주소는 필수입니다.",
            regex: "유효하지 않은 이메일 형식입니다."
        }
    },
    password: {
        required: true,
        minLength: 8
    }
    // ... passwordConfirmation 등 추가
};

```

### 2. 핵심 유효성 검사 코어 개발

1. **메인 함수:** 다음 시그니처를 가진 메인 검사 함수를 작성합니다: `validateForm(formElement, schema)`
2. **폼 데이터 수집:** `formElement`를 순회하며 각 입력 필드의 `name`과 `value`를 추출하여 검사에 사용할 수 있는 형태로 만듭니다.
3. **규칙 기반 검사:**
    - **`required`:** 입력 값이 비어있는지 확인.
    - **`minLength` / `maxLength`:** 문자열 길이 검사.
    - **`regex`:** 스키마에 정의된 정규식을 사용하여 값의 형식 검사.
4. **에러 객체 반환:** 검사가 실패한 모든 필드와 해당 에러 메시지를 포함하는 객체를 반환합니다.

### 3. 에러 메시지 렌더링 및 UI 업데이트

1. **UI 동기화:** `validateForm`이 반환한 에러 객체를 기반으로 UI를 업데이트하는 함수를 작성합니다.
2. **에러 표시:** 유효성 검사 실패 시, 해당 입력 필드 바로 아래에 스키마에 정의된 에러 메시지(예: `"최소 5자 이상 입력해야 합니다."`)를 표시합니다.
3. **성공 처리:** 모든 검사를 통과했을 경우, 폼 제출 로직을 실행하고 성공 메시지를 콘솔에 출력합니다.

### 🌟 심화 도전 과제 (Challenge Up)

1. **커스텀 규칙 등록:** `registerRule(ruleName, validationFunction)`과 같이 외부에서 새로운 검사 규칙을 추가할 수 있는 기능을 구현하여 라이브러리의 확장성을 높입니다.
2. **비밀번호 일치 확인:** `password` 필드와 `passwordConfirmation` 필드가 일치하는지 확인하는 **필드 간 비교(Cross-Field Validation)** 규칙을 구현합니다.
3. **디바운스 검사:** 사용자가 입력 중일 때(on `keyup` 또는 `input` 이벤트), 폼 제출 전에 실시간으로 유효성을 검사하되, **`debounce` 함수**를 사용하여 검사 횟수를 제한합니다.
