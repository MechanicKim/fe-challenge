# 11주차 챌린지 가이드: 아코디언(Accordion) 컴포넌트

11주차 챌린지는 정보를 구조적으로 정리하여 사용자 경험을 개선하는 **아코디언(Accordion) 컴포넌트**를 개발합니다. 이 과제는 단순한 토글 기능을 넘어, **부드러운 애니메이션 구현**과 **접근성 표준(WAI-ARIA)** 준수에 초점을 맞춥니다.

### 🚀 챌린지 개요: 목표와 요구사항

| 카테고리 | 상세 목표 |
| :--- | :--- |
| **핵심 기능** | 여러 섹션을 포함하며, 클릭 시 해당 섹션만 확장/축소되는 기능을 구현합니다. |
| **주요 기술** | **Vanilla JS 또는 프레임워크** (선택), CSS Transition, `scrollHeight`, WAI-ARIA 속성. |
| **애니메이션** | 콘텐츠가 펼쳐지거나 접힐 때 **높이 변화(Height Transition)**가 끊김 없이 부드럽게 보이도록 구현합니다. |
| **확장성** | 한 번에 **하나의 섹션만 열리도록 제한**할 수 있는 옵션(싱글 토글 모드)과 **여러 섹션이 동시에 열리는** 모드를 지원해야 합니다. |

---

### ⚙️ 필수 구현 단계

#### 1. HTML 구조 및 스타일

1.  **구조:** `<details>`와 `<summary>` 태그 대신 일반 `<div>`를 사용하여 아코디언 컴포넌트 구조를 설계합니다. 각 항목은 **헤더(버튼 역할)**와 **콘텐츠 패널**로 구성합니다.
2.  **초기 상태:** 모든 콘텐츠 패널은 기본적으로 숨겨져 있어야 합니다.

#### 2. 토글 로직 및 상태 관리

1.  **상태 관리:** 현재 열려 있는 패널의 `id` 또는 `index`를 저장하는 상태(State)를 관리합니다.
2.  **토글 함수:** 헤더 클릭 시 해당 패널의 상태를 토글하는 함수를 구현합니다.
3.  **싱글 토글 모드:**
    * 컴포넌트 초기화 시 옵션(예: `multiple: false`)을 받아 싱글 토글 모드를 결정합니다.
    * 이 모드에서는 새로운 패널이 열릴 때 **현재 열려 있는 다른 모든 패널을 자동으로 닫아야** 합니다.

#### 3. 부드러운 애니메이션 (핵심)

1.  **CSS `max-height` / `height` 문제 해결:** CSS `transition`을 `height: auto`에는 적용할 수 없으므로, 다음 방법을 사용합니다.
    * 닫힌 상태: `height: 0; overflow: hidden;`
    * 열린 상태: JavaScript를 사용하여 콘텐츠 패널의 실제 높이(`element.scrollHeight`)를 측정한 후, 해당 값을 CSS `height` 속성에 적용하고 `transition`을 활성화하여 부드러운 애니메이션을 구현합니다.
    * 애니메이션 완료 후: `height` 속성을 다시 `auto`로 설정하여 콘텐츠 변경에 대비합니다.

#### 4. 접근성 (WAI-ARIA) 준수

1.  **역할 및 상태:**
    * 헤더 버튼에 `role="button"`을 적용합니다.
    * 콘텐츠 패널에 `role="region"`을 적용합니다.
2.  **연결 및 상태 표시:**
    * 헤더 버튼에 `aria-expanded="true"` (열림) 또는 `aria-expanded="false"` (닫힘) 상태를 동적으로 업데이트합니다.
    * 콘텐츠 패널에 `aria-labelledby` 속성을 사용하여 해당 패널을 여는 헤더와 연결합니다.

### 🌟 심화 도전 과제 (Challenge Up)

실력을 더 높이고 싶다면 다음 항목에 도전해 보세요.

1.  **키보드 탐색:** `Up` 및 `Down` 화살표 키로 아코디언 헤더를 탐색하고, `Space` 또는 `Enter` 키로 확장/축소할 수 있도록 구현합니다.
2.  **지연 로딩:** 아코디언 패널의 콘텐츠가 **패널이 열리는 시점에만** 비동기적으로 로드되도록 시뮬레이션하고 구현합니다.
3.  **애니메이션 최적화:** `height` 속성 대신 `transform: translateY()`를 사용하여 애니메이션 성능을 개선할 수 있는 방법을 탐구하고 적용합니다. (다만 높이 애니메이션을 구현하는 주 목표에 집중하는 것이 우선입니다.)
