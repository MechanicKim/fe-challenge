# 8주차 챌린지 가이드: Next/Nuxt 기반의 고성능 랜딩 페이지 목업 구현

8주차 챌린지는 이전 챌린지들에서 습득한 모든 기술을 통합하여, 실제 상용 서비스에서 요구하는 **최상위 성능과 검색 엔진 최적화(SEO)**를 갖춘 웹 애플리케이션 아키텍처를 경험하는 데 초점을 맞춥니다.

주요 목표는 SSR/SSG 프레임워크를 사용하여 **Core Web Vitals(주요 웹 지표)**를 준수하는 랜딩 페이지 목업을 구축하는 것입니다.

### 🚀 챌린지 개요: 목표와 요구사항

| 카테고리 | 상세 목표 |
| --- | --- |
| **핵심 기술** | **Next.js** (React) 또는 **Nuxt.js** (Vue) 중 하나를 선택하여 프로젝트를 시작합니다. |
| **아키텍처** | **서버 사이드 렌더링(SSR) 또는 정적 사이트 생성(SSG)** 기능을 활용하여 초기 로딩 성능을 극대화합니다. |
| **성능 최적화** | LCP (Largest Contentful Paint) 및 CLS (Cumulative Layout Shift)를 최소화하기 위한 전략을 적용합니다. |
| **SEO** | 페이지 제목, 설명, Open Graph 태그 등 **메타데이터를 동적**으로 설정하여 SEO 기반을 마련합니다. |

### ⚙️ 필수 구현 단계

### 1. 프로젝트 설정 및 기본 레이아웃

1. **프레임워크 초기화:** 선택한 프레임워크(Next.js/Nuxt.js)의 기본 프로젝트를 생성합니다.
2. **랜딩 페이지 구조:** 일반적인 SaaS(Software as a Service) 랜딩 페이지 목업을 구현합니다. 최소한 다음 섹션을 포함해야 합니다.
    - **Hero Section:** 눈에 띄는 큰 제목과 CTA(Call to Action) 버튼.
    - **Features Section:** 제품의 주요 기능 3~4가지를 시각 자료와 함께 설명.
    - **Pricing/Testimonials Section:** 가격표 목업 또는 가상의 고객 후기.
    - **Footer:** 저작권 및 내비게이션 링크.

### 2. 성능 최적화 (Core Web Vitals 준수) (핵심)

1. **이미지 최적화:** 프레임워크가 제공하는 `<Image>` 컴포넌트(Next.js의 `next/image` 등)를 사용하여 이미지를 최적화하고, **레이지 로딩(Lazy Loading)**을 적용합니다.
2. **LCP 최적화:** Hero Section에 사용되는 가장 큰 콘텐츠 요소(이미지 또는 텍스트)가 빠르게 로드되도록 **`priority` 속성**을 사용하거나 최적화 전략을 적용합니다.
3. **폰트 최적화:** 외부 웹 폰트(Google Fonts 등)를 사용할 경우, **폰트 로딩 최적화** 기능을 사용하여 CLS 발생을 방지하고 빠르게 로드되도록 합니다.
4. **CSS/JS 분할:** 사용하지 않는 CSS 및 JavaScript 코드가 번들에 포함되지 않도록 모듈화하고, 프레임워크의 코드 분할 기능을 활용합니다.

### 3. 메타데이터 및 SEO 구현

1. **동적 `<head>` 태그 관리:** 프레임워크의 API(Next.js의 `next/head` 또는 Nuxt.js의 `useHead` 등)를 사용하여 다음 메타데이터를 설정합니다.
    - 페이지 제목 (`<title>`)
    - 페이지 설명 (`<meta name="description">`)
    - **Open Graph (OG) 태그:** 소셜 미디어 공유 시 사용될 제목, 설명, 이미지 URL 목업을 설정합니다.

### 4. 데이터 로딩 시뮬레이션

1. **SSR/SSG 활용:** 페이지의 주요 콘텐츠(예: Features Section의 텍스트)를 `getStaticProps` (SSG) 또는 `getServerSideProps` (SSR)와 같은 데이터 가져오기 함수를 사용하여 **빌드 타임 또는 요청 시 로드되는 것처럼 시뮬레이션**합니다. (실제 API 호출 대신 가상의 JSON 데이터를 사용해도 무방합니다.)

### 🌟 심화 도전 과제 (Challenge Up)

실력을 최상으로 올리고 싶다면 다음 항목에 도전해 보세요.

1. **인터랙션 관찰 (Intersection Observer):** 스크롤 시 섹션이 뷰포트에 들어올 때 애니메이션을 트리거하는 `Intersection Observer` API를 사용하여 사용자 경험을 개선합니다.
2. **라이트하우스 분석:** 완성된 목업을 Google Lighthouse 도구로 분석하고, **Performance 점수 90점 이상**을 달성하도록 최적화합니다.
3. **상태 관리 통합:** 1주차에서 만든 **대시보드 위젯**이나 4주차의 **테이블 컴포넌트** 중 하나를 통합하고, 전역 상태 관리 패턴을 적용하여 데이터 동기화를 시뮬레이션합니다.
